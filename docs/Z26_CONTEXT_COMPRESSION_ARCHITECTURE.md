# Z26 — Context Compression Architecture

## Purpose

Prevent long-session token explosion and context degradation without introducing fake AGI memory. This is runtime context management only.

## Architecture

### Context Layout (outermost → innermost in LLM prompt)

```
[System Prompt]
     ↓
[Critical Notes]        — retained indefinitely, hard-capped
     ↓
[Episode Summaries]     — compressed prior history (rolling)
     ↓
[Active Window]         — recent verbatim messages
```

### Components

**`SessionContext`** (`runtime/context_compression.py`)
- One instance per active session
- Tracks active window, episode summaries, and critical notes
- Fires auto-compression when token budget is exceeded

**Active Window**
- Fixed-size deque of verbatim `ContextMessage` objects
- Default: 40 messages
- When full, oldest half is compressed into an episode summary

**Episode Summaries**
- Rolling deque of `EpisodeSummary` objects
- Default budget: 8 episodes
- Each summary includes provenance hash for replay verification
- Can be generated by an LLM summarizer or a plain-text rollup fallback

**Critical Notes**
- Operator-tagged facts that must survive all compression cycles
- Types: `error`, `decision`, `constraint`, `goal`
- Hard cap: 20 notes (oldest evicted when exceeded)

## Token Budget

| Tier                | Default       |
|---------------------|---------------|
| Total budget        | 28,000 tokens |
| Active window       | ~40 messages  |
| Episode summaries   | 8 × ~200 tok  |
| Critical notes      | 20 × ~50 tok  |

Auto-compression fires when `total_tokens > token_budget`.

## Compression Flow

```
add_message()
    → token count exceeds budget?
        → compress()
            → take oldest 50% of active window
            → call summarizer_fn() or plain_rollup()
            → store EpisodeSummary with provenance_hash
            → replace active window with remaining 50%
            → audit log entry written
```

## Replay Safety

Each `EpisodeSummary` stores a `provenance_hash` derived from the SHA-256 of its source message IDs. `verify_replay_compatibility()` recomputes and compares this hash, flagging any integrity mismatch.

## Telemetry

`compression_telemetry()` returns per-session stats and audit log size. The audit log stores up to 500 entries with event type, timestamp, and session ID.

## FUTURE_RUNTIME markers

- `FUTURE_RUNTIME_RETRIEVAL`: semantic recall (chromadb) may be added as an injected retriever — never coupled directly to this module
- `FUTURE_RUNTIME_PERSISTENCE`: episode summaries should be persisted to SQLite before production multi-user deployment
- `FUTURE_RUNTIME_SUMMARIZER`: LLM-based summarization should be injected as a stateless callable, not hardwired
