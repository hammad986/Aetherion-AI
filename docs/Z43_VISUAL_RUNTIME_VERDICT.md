### Comparison: Before vs After Z43

**Before Z43:**
- Workspace felt static — no signal when AI is running vs idle
- Panels had mismatched padding, heights, border styles
- Context bar rendered as empty block when unused
- Run button was indistinguishable between idle and running states
- Topbar was visually disconnected from execution state
- Left rail icons had no active state treatment
- Empty states had no operational language
- Statusbar height was implicit, could expand unexpectedly
- Composer used hardcoded colors disconnected from token system
- Model button was large, unstyled, visually heavy

**After Z43:**
- Execution state signals propagate across 8 UI zones automatically
- All panels share a single spacing/border/elevation rhythm
- Context bar hidden when empty
- Run button clearly communicates idle (grey) vs running (blue + pulse)
- Topbar bottom border shifts to blue during execution
- Left rail has clean active state with left-edge indicator
- Empty states show operational guidance in mono forensic text
- Statusbar pinned at 22px — predictable, stable
- Composer uses full Z43/Z42 token system
- Model button compact, information-dense, execution-aware

### Honest Remaining Weaknesses

1. **Runtime mode granularity**: Only `idle` / `running` are wired. `paused`, `error`, `queued` modes are hooked in CSS but not yet wired in JS. Requires a single JS addition to `nx-z43-exec-state.js`.

2. **Monaco + xterm surfaces**: Both inject their own theme systems and cannot be governed by Z43 CSS tokens. Their surfaces are visually distinct from the workspace — acceptable for professional IDEs.

3. **Phase 8 modal density**: Billing modal uses inline styles — cannot be fully token-governed without touching inline HTML. Non-blocking for main workspace flows.

4. **DAG/timeline canvas**: Rendered via JS canvas — no CSS-level governance possible. Visual consistency within those surfaces is maintained by their own rendering code.

5. **Session age wiring**: `--z43-age-*` tokens defined; `data-session-age` attribute not yet set by JS. Implementation is < 10 lines of JS — deferred to Z44.
